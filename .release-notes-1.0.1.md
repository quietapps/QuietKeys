Sounds survive sleep and the lock screen.

## Fixed

- **Silent after sleep or lock** — waking the Mac or unlocking the screen left Quiet Keys silent until it was toggled off and on. The audio engine now rebuilds itself when CoreAudio tears it down, and the keystroke listener re-arms if macOS disabled it while the session was locked
- **Silent after switching audio devices** — changing the output device (e.g. connecting AirPods) now restarts the engine on the new device automatically

## Install

```sh
brew tap quietapps/quietkeys
brew install --cask quietkeys
```

Or download `QuietKeys-1.0.1.zip` below, drag **Quiet Keys.app** to `/Applications`, and run `xattr -cr "/Applications/Quiet Keys.app"` once.

Requires macOS 13 Ventura or later. See the [CHANGELOG](https://github.com/quietapps/QuietKeys/blob/main/CHANGELOG.md) for full notes.
